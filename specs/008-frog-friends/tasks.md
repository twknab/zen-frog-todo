## Phase 3: User Story 1 - Frogs gather as you get things done (Priority: P1) 🎯 MVP

**Goal**: Completing work (task +1, session +2, frog +3) makes frogs visibly gather around the bonsai base, bounded at ~20, in stable seeded positions that grow without reshuffling.

**Independent Test**: From a fresh day, complete a task / session / frog and watch 1 / 2 / 3 frogs appear around the pot; reload and confirm the arrangement is identical; keep going and confirm it stops at the cap.

- [x] T004 [P] [US1] In `src/components/BonsaiTree.tsx`: add a module-level `FROG_POSITIONS` array (length `MAX_FROGS`) of `{ x, y, scale }` slots clustered along the ground band around the pot base, generated by a deterministic index hash (no `Math.random` at render), with index 0 = the existing baseline frog spot.
- [x] T005 [US1] In `src/components/BonsaiTree.tsx`: add a `frogs: number` prop and render `FROG_POSITIONS.slice(0, clamp(frogs, BASELINE_FROGS, MAX_FROGS))` as small frog SVG groups (reuse the existing frog shape) inside `AnimatePresence` using the existing opacity-only `appear` variant; keep all critters in an `aria-hidden` layer and leave the `role="img"` label unchanged. **This SUBSUMES the existing hard-coded single frog (`translate(30 187)`) as `FROG_POSITIONS[0]` — do not render two baseline frogs (per analyze F2).** **Render the critter layer OUTSIDE the wilt treatment so frogs/squirrel stay full-color when the tree wilts (per analyze F1 / FR-007): apply the `isWilting` opacity+saturate styling only to the tree/pot layer, not to the critters** — restructure the wrapping `Box`/`svg` so the wilt filter no longer covers the critter group. Depends on T004.
- [x] T006 [US1] In `src/app/page.tsx`: pass `frogs={bonsai.frogs}` to `<BonsaiTree>` (same path as `leaves`/`blossoms`/`stage`). Depends on T002, T005.

**Checkpoint**: Frogs gather with rewards, bounded and stable — MVP functional and independently testable.

## Phase 4: User Story 2 - A fresh day, a fresh scene (Priority: P2)

**Goal**: Starting a new day returns the scene to the single baseline frog, consistent with the bonsai reset.

**Independent Test**: Accumulate frogs, start a new day, confirm the scene returns to one baseline frog (and the bonsai to a shrub); leave idle past midnight and confirm frogs persist until a manual new day.

- [x] T007 [US2] Confirm the frog population clears to baseline on "start a new day": because `frogs` derives from the event log that feature 007's `resetBonsai()` already clears, an empty log yields `BASELINE_FROGS`. Verify no separate frog state persists across the reset (no code expected beyond T002; add a guard/comment if any lingering source is found). Depends on T002.

**Checkpoint**: Day reset returns frogs to baseline; US1 + US2 both work.

## Phase 5: User Story 3 - An occasional squirrel (Priority: P3)

**Goal**: A single squirrel appears occasionally (deterministic on the frog count), gently, never more than one.

**Independent Test**: Step the frog count through several values and confirm the squirrel appears at some counts, not others, only one at a time, entering gently; reload at a given count shows the same state.

- [x] T008 [P] [US3] In `src/components/BonsaiTree.tsx`: add a pure `squirrelVisible(frogCount)` rule (`frogCount >= SQUIRREL_MIN && seededHash(frogCount) hits`) and a fixed seeded squirrel slot distinct from the frog slots (not counted in `MAX_FROGS`).
- [x] T009 [US3] In `src/components/BonsaiTree.tsx`: render a squirrel SVG conditionally on `squirrelVisible(frogs)` — at most one, gentle opacity entrance, `aria-hidden`. Depends on T008, T005.
