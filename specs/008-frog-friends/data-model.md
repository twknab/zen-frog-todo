# Data Model: Frog Friends

No new persisted key. The feature extends the existing bonsai event log and adds derived, non-persisted render inputs.

## Changed persisted shape

### GrowthEvent (`frog-garden:bonsai-v3`) â€” additive field

```ts
type GrowthEvent = {
  at: string;      // ISO-8601 (unchanged)
  leaves: number;  // leaf weight (unchanged: task=1, session=3, frog=5)
  frogs: number;   // NEW â€” frog weight: task=1, session=2, frog=3
};
```

- **Additive & back-compat**: no key version bump. Readers use `e.frogs ?? 0`, so events written before this feature contribute 0 frogs and nothing crashes on upgrade.
- Written by `recordGrowth(leaves, frogs)`; cleared with the rest of the log by `resetBonsai()` on "start a new day".

## New tunable constants (`src/lib/bonsai.ts`)

| Constant | Value | Purpose |
|---|---|---|
| `TASK_FROGS` | `1` | frogs per completed regular task |
| `SESSION_FROGS` | `2` | frogs per focus session |
| `FROG_FROGS` | `3` | frogs per completing the day's frog |
| `MAX_FROGS` | `20` | bounded crowd (incl. baseline) |
| `BASELINE_FROGS` | `1` | always-present lone frog (index 0) |
| `SQUIRREL_MIN` | `~6` | min crowd before a squirrel can appear (tunable) |

## Derived (not stored)

### Frog count â€” added to `BonsaiResult`

```ts

  stage: BonsaiStage;
  leaves: number;
  blossoms: number;
  isWilting: boolean;
  frogs: number;   // NEW

```

```

```

**Invariants:**

- Frogs do **not** subtract wilt â€” idle never reduces the frog count (unlike `leaves`).
- Day-scoped: because it sums the current event log (cleared on new day), a fresh day yields exactly `BASELINE_FROGS`.
- Not displayed as a number anywhere â€” drives rendering only.

### Frog placement â€” `FROG_POSITIONS` (module-level, computed once) in `BonsaiTree.tsx`

An array of length `MAX_FROGS` of `{ x, y, scale }` slots, clustered along the ground band around the pot base, generated by a deterministic index hash (no `Math.random` at render). Index 0 = the existing baseline frog spot. The component renders `FROG_POSITIONS.slice(0, frogs)`; frog `i` always occupies slot `i` (additive, stable, SSR-safe).

### Squirrel visibility â€” pure rule in `BonsaiTree.tsx`

```
squirrelVisible(frogCount) = frogCount >= SQUIRREL_MIN && seededHash(frogCount) hits
```

- Deterministic for a given `frogCount` (stable across re-render/SSR).
- Occasional (roughly one-in-N counts), never more than one squirrel.
- Rendered in its own fixed seeded slot, distinct from the frog slots and not counted in `MAX_FROGS`.

## Component prop change

`BonsaiTree` gains `frogs: number` (passed from `page.tsx` as `bonsai.frogs`, alongside the existing `leaves` / `blossoms` / `stage` / `isWilting` props). The squirrel is derived inside the component from `frogs`, keeping the component a pure render of props.
